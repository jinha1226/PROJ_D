The project's 1st goal is to **replicate Dungeon Crawl Stone Soup (DCSS) as faithfully as possible on mobile**, layered with **Pixel Dungeon's mobile-friendly UX conveniences** (touch controls, tap-to-move, streamlined UI).

**Why:** This is the user's guiding vision stated on 2026-04-20. Earlier simplifications (e.g., collapsing DCSS's ~30 skills to melee/ranged/magic/support) run counter to this goal and should be reversed.

**How to apply:**
- When in doubt between "simpler" and "DCSS-accurate", pick DCSS-accurate.
- Balance, map generation, skill system, spell behavior, monsters, items — default to matching DCSS 0.34 mechanics and names.
- UX layer (controls, inventory screen, quickslots, auto-explore, combat feedback, touch interactions): apply Pixel Dungeon mobile idioms.
- Push back on proposals that simplify DCSS away ("let's combine these three skills"), unless the user explicitly asks for that simplification.
- When the user asks to "restore" or "un-simplify" something, assume they want the DCSS-aligned version.
